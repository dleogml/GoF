## 빌더(Builder) 패턴
- 동일한 프로세스를 거쳐 다양한 구성의 인스턴스를 만드는 방법
- (복잡한) 객체를 만드는 프로세스를 독립적으로 분리할 수 있다

### 빌더 패턴의 장단점
- 장점
  - 복잡한 객체를 순차적으로 만들 수 있게 해줌
  - 구체적인 과정을 어느정도 숨겨줌
  - 불완전한 객체를 사용하지 못하게 막아줌
- 단점
  - 빌더(또는 디렉터) 인스턴스를 먼저 만들어야 함
  - 구조는 복잡해질 수 있음
